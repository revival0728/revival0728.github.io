KMP algorithm

Concept

The time complexity of this algorithm is \(O(N+M)\),

which \(N\) is the length of text and \(M\) is the lenght of pattern.

Before doing this algorithm, we have to do some preprocesses.

We find the LPS(Longest prefix which is also Suffix) of the pattern string.

The LPS will help us not to search the substring of text which is in the pattern string again.

After finished building LPS, we can start searching.

The searching step is kind of naive.

Use naive search until you find a character is not same.

When you find a character is not same, you move pattern string's index to the last LPS of this index.

And keep doing this until the index of text string is equal to its length.
