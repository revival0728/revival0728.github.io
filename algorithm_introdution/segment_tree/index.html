Segment Tree

Concept

Segment tree is a data structure for range querries.

When a data can process by segment tree.

This data must be mergeable.

For example sum querries, multiplication querries, XOR querries, etc.

When you trying to build a segment tree.

You have to code three functions. build(), update(), and querry()

The time complexity of build() is \(O(N)\).

The update()'s is \(O(\log N)\).

And the querry()'s is \(O(\log N)\).

The total space complexity is \(O(4\times N)\) which is \(O(N)\).

Usually we use recursion to implement the segment tree.

Single Update

The implement of single update segment tree is kind of naive.

Basically, just recursion until you find the range you querry.

The way to recursion is keep cutting the segment into half until we find the range or the length of segment becomes one.

When you are recursing, just be careful about the border of segment tree.

If you reach to bottom of segment tree, you cannot pull anything.

Or you will get a segment error.
