Ternary Search

Concept

This is a search algorithm which has \( O(\log_3 N) \) time complexity.

The different between binary search is it has 2 middle value when searching.

When searching, we cut the segment into 3 part which has same length.

  1. Left to first middle
  2. First middle to second middle
  3. Second middle to right

Just like binary search. We analyze every part of segment.

First hypothesis we are searching value key in array arr[N].

Following is the recursive relationship for this searching.

\( \begin{cases} If \quad key < arr[Mid_L], \quad R = Mid_L-1 \\ If \quad key > arr[Mid_R], \quad L = Mid_R+1 \\ Else \quad L = Mid_L+1, \quad R = Mid_R-1 \end{cases} \)

Know we know how to search the value.

But how to calculate the value of \(Mid_L\) and \(Mid_R \) ?

Because the ternary search cut the segment into 3 part which has same length.

So the \( Mid_L=l+\frac{r-l}{3} \) and the \(Mid_R=r-\frac{r-l}{3} \)

Now we know the concept of ternary search.

That means we can start coding!
